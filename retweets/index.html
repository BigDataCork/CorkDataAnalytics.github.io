Who is retweeting tweets about '@CSOIreland'?

Interesting question….this is how we find the answer.

First you need to be able to identify yourself as a registered twitter user to twitter. More details on that are on the link below.

http://thinktostart.com/twitter-authentification-with-r/

After you have done that you can use the code below. This code is valid in June 2015, however as R updates quite frequently code instructions and functions can become “deprecated”. The term deprecated means that the code you have entered has been tweaked and no longer works with your version of R, is no longer used or has been renamed. library("twitteR")
library("igraph")
library("stringr")
#Harvest tweets about "CSOIreland"
# tweets in english containing "CSOIreland"
dm_tweets = searchTwitter("@CSOIreland", n=1000, lang="en") 
# get text
dm_txt = sapply(dm_tweets, function(x) x$getText())


# regular expressions to find retweets
grep("(RT|via)((?:\\b\\W*@\\w+)+)", dm_tweets, 
     ignore.case=TRUE, value=TRUE)

# which tweets are retweets
rt_patterns = grep("(RT|via)((?:\\b\\W*@\\w+)+)", 
                   dm_txt, ignore.case=TRUE)

# show retweets (these are the ones we want to focus on)
#dm_txt[rt_patterns] 


#Collect who retweeted and who posted
#We'll use these results to form an edge list in order to create the graph
# create list to store user names
who_retweet = as.list(1:length(rt_patterns))
who_post = as.list(1:length(rt_patterns))

# for loop
for (i in 1:length(rt_patterns))
{ 
  # get tweet with retweet entity
  twit = dm_tweets[[rt_patterns[i]]]
  # get retweet source 
  poster = str_extract_all(twit$getText(),
                           "(RT|via)((?:\\b\\W*@\\w+)+)") 
  #remove ':'
  poster = gsub(":", "", unlist(poster)) 
  # name of retweeted user
  who_post[[i]] = gsub("(RT @|via @)", "", poster, ignore.case=TRUE) 
  # name of retweeting user 
  who_retweet[[i]] = rep(twit$getScreenName(), length(poster)) 
}

# unlist
who_post = unlist(who_post)
who_retweet = unlist(who_retweet)

# Create graph from an edglist
# two column matrix of edges
retweeter_poster = cbind(who_retweet, who_post)

# generate graph
rt_graph = graph.edgelist(retweeter_poster)

# get vertex names
ver_labs = get.vertex.attribute(rt_graph, "name", index=V(rt_graph))

# Let's plot the graph
# choose some layout
glay = layout.fruchterman.reingold(rt_graph)

# plot
par(bg="gray15", mar=c(1,1,1,1))
plot(rt_graph, layout=glay,
   vertex.color="gray25",
   vertex.size=15,
   vertex.label=ver_labs,
   vertex.label.family="sans",
   vertex.shape="none",
   vertex.label.color=hsv(h=0, s=0, v=.95, alpha=0.5),
   vertex.label.cex=0.5,
   edge.arrow.size=0.5,
   edge.arrow.width=0.2,
   edge.width=5,
   edge.color=hsv(h=.95, s=1, v=.7, alpha=0.5))
# add title
title("Tweets with '@CSOIreland':  Who retweets whom",
   cex.main=1, col.main="gray95")

We can see that CSOIreland is the main source of retweets with traffic being largely one way, i.e. users mainly retweet CSOIreland and it does not retweet those same users (if they refer to it). Conversely, we can also see that CSOIreland has retweeted one user who has not retweeted one of it's tweets. Also on the graph we see that there are several other centres of information. While traffic is not as large, it is clear that people are reading tweets that contain CSOIreland which are not retweets from CSOIreland. In other words there is evidence that tweeters are using these other centres for news on CSOIreland rather than visit the CSOIreland twitter page. This news may simply be a reiteration of what's posted on CSOIreland or the CSO website, or it may be some commentary that refers to CSOIreland.

Now let’s have a look at a Hierarchical Agglomerative cluster plot/dendogram of words clustered by the frequency in which they occur and how they are related to other frequently occuring terms. Their proximity is calulated by an algorithm using euclidean distance to determine the nearest term.

To make the dendogram, the number of terms is reduced, that is to say the sparse terms (those occurring less than 5% of the time) are removed, because all terms would not all fit into the page.

Terms higher in the plot appear more frequently within the body of tweets.

Terms grouped near to each other are more frequently found together.

In dendogram we can see that CSOIreland is the most frequent term and its nearest neighbours in terms of frequency are 'may' and 'unemployment'. Likewise 'may' and 'unemployment' are more frequently found together.

Now we can build a network graph that represents the number of co-occurrences of two terms/words, i.e. a term and itself or another term.

This is done by taking the data.frame used to construct the dendogram. This data.frame is converted to an adjacency matrix. As the original data.frame has 37 rows and 80 observations a square version of this data.frame (let's say first 30 rows and first 30 observations) is needed to make the adjacency matrix (i.e. n x n). This square data.frame is first converted into a Boolean matrix (i.e. frequencies of 1 and 0). The Boolean matrix is then converted into the adjacency matrix which gives the co-occurence frequencies for any given word ( a word will co-occur with itself as often as it occurs, e.g. if it appears 37 times then it has co-occurred 37 times) or pair of words. The adjacency matrix is then used to plot the graph below. However, as the adjacency matrix is smaller than the original data.frame and the data used to make the dendogram, we can expect the results to be different.

We can see from this graph that CSOIreland occurs most and other than itself, most frequently co-occurs with 'may' followed by 'rate' and 'now'. Although 'may' and 'unemployment' appear in the dendogram to be most frequently found with CSOIreland, 'unemployment' does not appear in the network graph. This is because 'unemployment' was removed (observation number 36 in the data.frame) when the data.frame was converted to a square matrix. We can see that 'may' co-occurs (after CSOIreland) most closely with 'rate', 'now', 'live', 'allaboutjobs' and 'persons'. And so on.
